LangGraph adapter — schema-driven chaos against any compiled graph

Drift takes a compiled LangGraph (or anything with .invoke(state)), reads the runtime types of your state, generates type-appropriate perturbations (flip bool, remove dict key, reverse list, corrupt string, etc.), and invokes your graph once per perturbation. Results bucket into crashed, diverged from baseline, or unchanged.

Demo graph below is a 3-node ticket triage that looks up open_tickets[ticket_id] in both terminal nodes without a defensive check — exactly the kind of bug a developer wouldn't write a test for. Chaos that empties or rekeys the dict reliably crashes it.

Substring match against pattern names. flip_bool skips all bool flips; open_tickets skips every perturbation targeting that field.

Bring your own agents + environment

Define your multi-agent system in the editor below. Four optional pieces drift looks for:

  • Environment — subclass drift.WorldState with your own fields (orders, inventory, security status, whatever your domain needs)
  • Events — subclass drift.Event with an apply(world) method that mutates state during a run (chaos injection)
  • initial_state() — function returning a populated state instance (drift uses an empty WorldState if absent)
  • events() — function returning [(timestep, event_instance), …] for scheduled chaos

Then define one or more @drift.agent async functions that take (state, memory) and return a drift.Action. Replace the function bodies with your real LLM/tool/framework calls — drift only needs the structured Action back so the detectors can read it.

Note: this executes Python in the local server process. Safe for local development; do not expose this server to untrusted users.

When on, drift inspects your WorldState fields and auto-injects type-appropriate mutations (bool flips, dict clears / fake keys, list reverses, numeric boundaries, string corruption) at random steps. Auto-events show up in the results list prefixed AutoChaos.<pattern>[field] so you can map fired failures back to specific injected chaos.

Adds an LLM-judged detector alongside the deterministic ones. Failures it reports are prefixed llm: across the five families (coordination_contradiction, grounding_failure, state_drift, emergent_decay, gate_bypass) so you can tell them apart from deterministic detections.

Plain-English patterns drift's judge will additionally watch for. Matches are reported under llm:user_guideline:<n> in the failures list. This is the programmable test surface — write what's specific to your domain (anything the five-family taxonomy doesn't cover). Empty = current behaviour.

About drift

Drift is a multi-agent stress-test simulator. It puts a small team of AI agents into a fake environment that changes over time, and watches for failures that only show up when agents work together — contradictions, loops, hallucinated references, coordination failures across timesteps.

This UI runs locally against the same Python codebase you'd use from the CLI. Runs are written to runs/ as JSONL and can be replayed or compared.
